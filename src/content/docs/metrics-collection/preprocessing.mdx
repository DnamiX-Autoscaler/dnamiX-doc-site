<DocSection id="overview" title="Overview">

Raw Prometheus metrics are not directly suitable for machine learning.

They may contain:

- Missing samples
- Different timestamps
- Different sampling intervals
- Noisy values
- High-cardinality labels
- Unnormalized scales

DnamiX preprocessing transforms this data into consistent ML-ready rows.

</DocSection>

<DocSection id="cleaning" title="Cleaning">

Cleaning removes unusable or invalid values.

<CodeBlock
  language="python"
  title="Basic cleaning logic"
  code={`def clean_value(value, default=0.0):
    if value is None:
        return default
    if value == "NaN":
        return default
    return float(value)`}
/>

</DocSection>

<DocSection id="alignment" title="Time Alignment">

All metrics are aligned into common time windows.

Example:

<CodeBlock
  language="text"
  code={`15s window:
10:00:00
10:00:15
10:00:30
10:00:45`}
/>

This makes node, pod, app, and mesh metrics comparable at the same timestamp.

</DocSection>

<DocSection id="normalization" title="Normalization">

Normalization converts metrics into similar numeric ranges.

<CodeBlock
  language="python"
  title="Min-max normalization"
  code={`def normalize(value, min_value, max_value):
    if max_value == min_value:
        return 0.0
    return (value - min_value) / (max_value - min_value)`}
/>

</DocSection>

<DocSection id="merging" title="Feature Merging">

After cleaning and alignment, metrics are merged by:

- timestamp
- namespace
- service name
- pod name, when required

<Alert type="info" title="Why preprocessing is important">
ML models cannot produce reliable predictions if the telemetry data is incomplete, noisy, or not time-aligned.
