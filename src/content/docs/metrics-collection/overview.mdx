<DocSection id="overview" title="Overview">

The Metrics Collection component is the data foundation layer of DnamiX.

It collects telemetry from:

- Node layer
- Pod/container layer
- Application layer
- Service mesh layer

The purpose is to transform raw Kubernetes monitoring data into a structured, topology-aware, ML-ready dataset.

</DocSection>

<DocSection id="layers" title="Telemetry Layers">

| Layer | Source | Example Metrics |
|---|---|---|
| Node | Node Exporter | CPU, memory, disk, network |
| Pod | cAdvisor / kube-state-metrics | pod CPU, pod memory, restarts |
| Application | prom-client | request rate, latency, error rate, queue length |
| Service Mesh | Istio / Envoy | service-to-service traffic, request duration |

</DocSection>
