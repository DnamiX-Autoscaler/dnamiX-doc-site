<DocSection id="overview" title="Overview">

Istio provides service-to-service telemetry required for graph construction.

DnamiX uses Istio metrics to identify:

- Source service
- Destination service
- Request rate
- Latency
- Error rate
- Traffic direction

</DocSection>

<DocSection id="podmonitor" title="Envoy Sidecar PodMonitor">

<CodeBlock
  language="yaml"
  title="03-istio-servicemonitors.yaml"
  code={`apiVersion: monitoring.coreos.com/v1
kind: PodMonitor
metadata:
  name: pm-envoy-sidecars
  namespace: monitoring
  labels:
    release: prometheus
spec:
  namespaceSelector:
    matchNames: ["ecommerce-prod"]
  selector:
    matchExpressions:
      - key: security.istio.io/tlsMode
        operator: Exists
  podMetricsEndpoints:
    - port: http-envoy-prom
      path: /stats/prometheus
      interval: 15s`}
/>

</DocSection>

<DocSection id="peer-auth" title="PeerAuthentication">

<CodeBlock
  language="yaml"
  title="05-istio-peer-auth.yaml"
  code={`apiVersion: security.istio.io/v1
kind: PeerAuthentication
metadata:
  name: default
  namespace: ecommerce-prod
spec:
  mtls:
    mode: PERMISSIVE`}
/>

<Alert type="warning" title="Development vs Production">
`PERMISSIVE` mode is useful during development because it allows Prometheus scraping. For strict production environments, review mTLS and scraping policies carefully.
</Alert>

</DocSection>

<DocSection id="graph-query" title="Service Graph Query">

<CodeBlock
  language="promql"
  title="Istio service-to-service request flow"
  code={`sum(rate(istio_requests_total{reporter="source"}[1m]))
by (source_workload, destination_workload)`}
/>

This query becomes the edge list for the dependency graph:

<CodeBlock
  language="text"
  code={`source_workload -> destination_workload`}
/>

</DocSection>
