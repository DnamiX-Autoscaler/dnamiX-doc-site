<DocSection id="overview" title="Overview">

Prometheus is the primary time-series database used by DnamiX.

It collects:

- Node-level metrics
- Pod/container metrics
- Application metrics
- Istio service mesh telemetry
- DnamiX component metrics

<DocSection id="values" title="kube-prometheus-stack Values">

<CodeBlock
  language="yaml"
  title="01-kube-prom-stack-values.yaml"
  code={`prometheus:
  prometheusSpec:
    scrapeInterval: 15s
    evaluationInterval: 15s

    serviceMonitorNamespaceSelector: {}
    podMonitorNamespaceSelector: {}

    serviceMonitorSelector:
      matchLabels:
        release: prometheus

    podMonitorSelector:
      matchLabels:
        release: prometheus

grafana:
  enabled: true
  adminPassword: admin
  service:
    type: ClusterIP

prometheus-node-exporter:
  hostRootFsMount:
    enabled: false`}
/>

</DocSection>

<DocSection id="why" title="Why This Configuration Matters">

This configuration allows Prometheus to discover ServiceMonitors and PodMonitors from the monitoring namespace and scrape application metrics every 15 seconds.

<Alert type="info" title="Windows Docker Desktop Note">
The `hostRootFsMount.enabled: false` setting avoids mount propagation issues in Docker Desktop and Kind environments.
</Alert>

</DocSection>
